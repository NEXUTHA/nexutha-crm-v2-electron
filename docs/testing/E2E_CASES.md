# E2Eテストケース定義

**作成**: 2026-07-20 / **実装予定**: v2.5.3（Playwright）
**目的**: 台帳（`docs/audit/REPAIR_LEDGER_2026-07.md`）の各項目が再発しないことを機械的に保証する。

現状 electron版に自動テストは**存在しません**（台帳 F-04）。旧V2の `tests/` は移植されていません。

## 実装メモ

- 旧V2の `tests/`（Playwright 14本）が参考になるが、**electron版はブラウザではなくElectronアプリ**のため `@playwright/test` の `_electron` を使う
- **テスト用DBを本番と分離**すること（旧V2は `com.nexutha.crm.test` 相当の分離を行っていた）
- **A-3〜A-5, B-1〜B-5（オフライン系）を最優先で実装**。これが今回の障害の再発防止に直結する
- K系はバックエンドのトークン認証実装（v2.5.3 / MCP連携ブランチ）とセットで
