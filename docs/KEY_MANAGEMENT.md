# ライセンス署名鍵の取扱説明書

**読む人**: 山下さん / **最終更新**: 2026-07-20

---

## この鍵は何か

NEXUTHA CRM のライセンスキーは、**「山下さんが発行したもので間違いない」ことを数学的に証明できる印**が押されています。その印を押すための道具が **署名鍵** です。

鍵は2つで1組です。

| | 役割 | 扱い |
|---|---|---|
| **秘密鍵** | ライセンスキーを**作る**ための鍵 | 🔴 **絶対に人に見せない。山下さんのMacの中だけ** |
| **公開鍵** | ライセンスキーが**本物か確かめる**ための鍵 | 🟢 アプリに入れて配布する。人に見せてOK |

公開鍵から秘密鍵を逆算することは**数学的にできません**。だからアプリを何万本配っても偽造されません。

---

## 一番大事なこと（3つだけ）

### 1. 🔴 秘密鍵が漏れると、誰でも無限にライセンスを作れます

漏れた場合は**必ず鍵を作り直してください**（`rotate`）。作り直すと、それまでに発行した全ライセンスキーが無効になり、お客様に新しいキーを送り直す必要があります。

### 2. 🔴 秘密鍵を失くすと、新しいライセンスを発行できなくなります

Macが壊れる・買い替える場合に備えて、**必ずバックアップを取ってください**（`backup`）。既存のお客様は使い続けられますが、**新規販売ができなくなります**。

### 3. 🟢 このツールは秘密鍵を画面に表示しません

**表示する機能そのものがありません。** 「控えるために表示する」という操作は不要ですし、できません。バックアップは暗号化されたファイルとして直接USBに書き出されます。

> 2026-07-20、秘密鍵を画面に表示させる手順が事故につながりました。**人が気をつける前提の手順は、いずれ必ず破られます。** そのため、表示機能を廃止しました。

## 使い方

### 今の状態を見る

```
cd ~/NEXUTHA-CRM-V2-electron
node tools/key.js status
```

鍵があるか、最後にバックアップした日はいつかが分かります。**鍵の中身は出ません。**

### はじめて鍵を作る

```
node tools/key.js init
```

新しい公開鍵が表示されます。これをアプリに埋め込みます（公開鍵なので貼って渡して大丈夫です）。

### 🔴 鍵を作り直す（漏れたとき）

```
node tools/key.js rotate
```

確認のため `rotate` と入力します（入力は表示されません）。

**作り直すと、これまでのライセンスキーは全部無効になります。** ただしアプリの「移行フラグ」により、**すでに認証済みのお客様はそのまま使い続けられます**。再インストールした場合だけ新しいキーが必要です。

### 💾 バックアップを取る（必ず実施）

USBメモリを挿してから：

```
node tools/key.js backup
```

- USBは自動で見つかります（複数ある場合は指定を求められます）
- **パスフレーズ（合言葉）**を2回入力します。入力中は画面に出ません
- 暗号化されたファイルがUSBに書き出されます。画面に出るのは**保存先のパスだけ**です

**保管方法（重要）**

| 物 | 置き場所 |
|---|---|
| USBメモリ | 金庫・鍵のかかる引き出しなど |
| パスフレーズを書いた紙 | **USBとは別の場所** |

⚠️ **パスフレーズを忘れると復元できません。** こちらでも復旧できません。必ず紙に書いてください。

USBが無い場合は保存先を指定できますが、**Mac本体に置くのは意味がありません**（Macが壊れたら一緒に失われます）。

```
node tools/key.js backup ~/Desktop     ← 一時的な確認用。必ずUSB等へ移すこと
```

### バックアップから復元する（Mac買い替え・故障時）

```
node tools/key.js restore /Volumes/USBの名前/nexutha-license-key-backup-2026-07-20.json
```

パスフレーズを入力すると、キーチェーンに鍵が戻ります。復元後に**公開鍵が表示される**ので、アプリに埋め込まれている値と同じか確認してください。同じなら、これまでのライセンスキーがそのまま使えます。

### ライセンスキーを発行する

```
node tools/make-license.js --type master                        # 自分用
node tools/make-license.js --type normal --name "〇〇株式会社"   # 販売用
node tools/make-license.js --type trial  --name "体験版" --days 30
```

- 発行のたびに `tools/licenses.csv`（発行台帳）に自動で追記されます
- `--name` は**手元の台帳に記録するだけ**で、キーには入りません（キーに個人情報を入れない設計）
- 誰のキーかは、キーに書かれた**通し番号**を台帳で引いて確認します

## やっていいこと / ダメなこと

| | |
|---|---|
| ✅ 公開鍵をチャットやメールに貼る | 公開鍵は見られても安全です |
| ✅ ライセンスキーをお客様にメールで送る | そのための物です |
| ✅ 暗号化バックアップファイルをUSBに置く | パスフレーズが無ければ開けません |
| 🔴 `security find-generic-password ... -w` を実行する | **秘密鍵が画面に出ます。実行しないでください** |
| 🔴 秘密鍵をチャット・メール・クラウドに貼る | 漏洩＝鍵の作り直しが必要になります |
| 🔴 `tools/licenses.csv` をGitHubに上げる | お客様の名前が入っています（`.gitignore` で除外済み） |
| 🔴 暗号化バックアップとパスフレーズを同じ場所に置く | 両方揃うと復元できてしまいます |

## 困ったときは

| 状況 | どうするか |
|---|---|
| **秘密鍵が漏れたかもしれない** | `node tools/key.js rotate` → 新しい公開鍵をアプリに埋め込み → 再ビルド → お客様に新キーを送付 |
| **Macを買い替える** | 旧Macで `backup` → 新Macで `restore` |
| **Macが壊れた** | 新Macで `restore`。バックアップが無い場合は `init` で作り直し（既存のお客様は移行フラグで継続利用可） |
| **パスフレーズを忘れた** | そのバックアップは使えません。鍵が生きているうちに `backup` を取り直してください |
| **鍵もバックアップも無い** | `init` で新規作成。既存のお客様は認証済みなら継続利用できます。再インストールした方には新キーが必要です |
| **バックアップを取ったか分からない** | `node tools/key.js status` |

## 補足：この仕組みの限界（正直に）

- **失効させられません。** オフラインで検証するため、発行済みキーを後から無効化できません。返金・解約時にキーを止められません。※ アプリ同梱の失効リスト（`revoked.json`）をアップデートで配る方法はありますが、**更新しないお客様には効きません**
- **台数制限がありません。** 同じキーで何台でも使えます
- **改造すれば回避できます。** お客様のMac上で検証するため、技術的な知識がある人は回避可能です。v1ではこれを許容しています（対策のコストが被害に見合わないため）
